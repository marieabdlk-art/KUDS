# Contributing

Thanks for your interest in KUDS.

This repository is currently a research prototype. The most useful contributions at this stage are:

- reproducibility fixes
- notebook cleanup
- clearer metric reporting
- better ablation design
- issue reports on pseudo-novelty and selector smoothing
- documentation improvements

## Before opening a pull request

Please try to:

1. describe the exact notebook or file affected
2. explain whether the change is conceptual, evaluative, or implementation-only
3. keep claims narrow and evidence-based
4. avoid adding new marketing claims without corresponding results

## Good first contributions

- improve notebook stability
- improve JSON repair / parsing
- add cleaner exports for pilot metrics
- document failure modes
- improve prompt formatting without changing evaluation semantics
